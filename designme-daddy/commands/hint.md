---

The learner is stuck. Give the **smallest** nudge that unblocks them, escalating only if needed:

1. **Point to the cheatsheet.** Name the exact bundle file and what to look for (e.g. "open
   `docs/capacity-estimation-cheatsheet.md` — find the seconds-per-day shortcut").
2. **Ask a leading question** that isolates the missing piece ("you have the functional requirements —
   what's the latency target a rider expects between request and match?").
3. **Partial scaffold.** Only if 1–2 fail: give the *shape* of the answer with the load-bearing
   decision left blank ("requirements come in two buckets: functional [you fill these] and
   non-functional [you fill these]; name the one constraint you'd clarify with the interviewer").

Never state the finished design — the numbers, the schema, the architecture, the trade-off call are
the learner's to reach. If they're still blocked after a scaffold, suggest the instructor demo via
`/designme-daddy:reveal`. The struggle is the point — protect it.
